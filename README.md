
# IP Weave

基于 GLM-5.1 的自治链上 IP 衍生内容 Agent。给一个链上 IP，它还你一个世界。

---

## 为什么做这个项目

链上 IP 资产在发行后普遍面临一个核心困境：mint 即终点。大量 NFT 项目在发售结束后缺乏持续的内容运营能力，社区成员想参与衍生创作但受限于专业技能，自发产出的内容质量参差不齐且风格难以统一。这导致大量链上 IP 陷入沉睡，其文化和商业价值远未被充分挖掘。

传统内容生产需要完整的创作团队：作家、分镜师、设计师、3D 建模师、开发者等。IP Weave 的目标是用一个 AI Agent 替代整个人类创作团队，让一个人加一个 Agent 就能完成从创意输入到链上交付的完整内容生产链路。

## 解决了什么问题

IP Weave 解决了链上 IP 生态中三个核心问题：

第一，内容生产门槛高。持有者想为 IP 创作衍生内容但缺乏写作、分镜、设计等专业技能。IP Weave 让用户只需要输入一个 IP 名称，Agent 自动完成全部创作。

第二，风格不统一。社区自发创作的内容风格参差，难以与原始 IP 对齐。IP Weave 通过 GLM-5.1 深度分析每个 IP 的独特风格，确保所有产出风格一致。

第三，缺乏变现路径。衍生内容创作后缺少上链变现的渠道。IP Weave 自动将内容打包为 NFT 元数据，生成 Solidity 合约，可一键部署到测试网。

## 为什么能做到自主决策

IP Weave 采用 ReAct 架构，GLM-5.1 在每个循环中分析当前状态并自主决定下一步动作。每轮循环中 Agent 会评估已经完成了什么、还缺什么、下一步最应该做什么。不是按固定顺序执行，而是真正的思考决策。

例如在处理 Azuki IP 时，Agent 的决策过程是：

循环 1: 当前数据为空,需要先读取链上数据 -> 调用 ip_reader
循环 2: 数据已读取,需要分析风格特征 -> 调用 analyze_style
循环 3: 风格已分析,需要创作故事 -> 调用 write_story
循环 4: 故事已完成,需要转为动画脚本 -> 调用 write_script
循环 5: 脚本已完成,需要设计资产 -> 调用 design_assets
循环 6: 所有内容生成完毕,需要检查质量 -> 调用 check_quality

每次决策基于当前实际状态,不是写死的流程。

## 核心功能

链上 IP 数据读取。自动读取任意 ERC-721 合约的链上数据,从以太坊主网实时获取 IP 的名称、描述、属性列表和 tokenURI 元数据。

IP 风格分析。GLM-5.1 提取 IP 独有的视觉风格、叙事基调、文化背景和代表符号。每个 IP 的分析结果都是独特的,包含 lore 世界观背景、character 角色性格习惯、key_symbols 代表性符号等信息。

衍生故事创作。基于 IP 独有的世界观和角色设定,用 GLM-5.1 创作高质量的衍生故事。完整的故事弧线包括起因发展转折高潮收束。篇幅 2000 到 4000 字。

动画分镜脚本生成。将故事转化为专业的动画分镜脚本,包含 6 到 8 个分镜。每个分镜详细描述场景画面、对白内容、镜头运动方式和持续时间。

周边资产概念设计。设计可上链的周边资产概念,每轮生成 2 款设计。每款资产包含产品名称、类型、详细设计描述、设计理念说明和 NFT 元数据。

风格一致性检查。GLM-5.1 评估生成内容与 IP 风格画像的匹配度。评分标准为 0 到 10 分,7 分以上为通过。未通过的内容自动触发迭代修正。

链上发布准备。将衍生内容打包为 NFT 元数据,生成 Solidity 合约代码。元数据使用 data URI 直接嵌入链上,不需要 IPFS。

3D 模型生成。集成混元 Hunyuan 3D API,支持根据文本描述生成 GLB 格式的 3D 模型,可用于数字穿戴和虚拟穿搭场景。

## 技术架构

IP Weave 采用两层架构：Agent 决策层和工具执行层。

Agent 决策层是系统的总指挥,基于 GLM-5.1 的分析能力运行 ReAct 循环。每轮循环中 Agent 执行三个动作:读取当前状态,分析还缺什么、哪些步骤已经完成、哪些需要改进;根据分析结果决定下一步调用哪个工具;执行工具调用后评估结果,判断是否需要迭代修正。

工具执行层是独立的功能模块,每个工具负责一个特定领域的工作。当前系统注册了六个工具:ip_reader 从以太坊主网读取链上数据,style_analyzer 用 GLM-5.1 分析风格指纹,story_writer 用 GLM-5.1 创作衍生故事,script_writer 将故事转为分镜脚本,asset_designer 设计周边资产概念,quality_checker 检查内容风格一致性。

整个架构的关键在于 GLM-5.1 在每个决策点都进行真实的分析和推理,而不是盲从固定流程。

## 使用到的 API SDK AI 工具

GLM-5.1 通过 Z.AI Coding Plan 调用。用途是 Agent 的核心大语言模型,负责所有决策、分析、创作和质量检查。使用位置包括 Agent 决策循环、风格分析、故事创作、脚本生成、资产设计、质量评估。效果能达到 3000 字以上的高质量叙事创作,能根据 IP 特征自主调整风格,能对生成内容进行多维度质量评估。

CogView-4 智谱 AI 图片生成。用途是周边资产概念图的生成。使用位置在 AssetDesigner 生成资产概念描述后调用 CogView-4 API 输出图片。效果将文本设计描述转化为可视化图片。

web3.py 用于以太坊链上数据读取和合约交互。使用位置在 OnChainIPReader 通过 web3.py 读取 ERC-721 合约的 name、symbol、tokenURI。效果实现对以太坊主网的实时数据读取。

混元 Hunyuan 3D 腾讯云 3D 生成。用途是周边资产 3D 模型生成,支持文本生成 GLB 格式模型。使用位置在 3D 生成模块通过 API 提交和查询任务。效果将文本描述转化为 3D 数字资产。

solc Solidity 编译器用于将 ERC-721 合约源码编译为 EVM 字节码。使用位置在合约部署流程中编译合约。效果支持动态编译和部署自定义 NFT 合约。

python-dotenv 用于从 .env 文件加载环境变量。使用位置在全局配置模块。效果安全管理 API Key。

## 安装与运行

前置条件:Python 3.10 或更高版本,GLM-5.1 API Key。

安装步骤:git clone 克隆仓库,cd 进入目录,pip install 安装依赖,配置 .env 文件填入 ZHIPUAI_API_KEY。

运行方式:python src/run_agent.py 然后输入 IP 名称;或 echo "bayc" | python src/run_agent.py 直接指定。
