#  IP Weave — 链上衍生宇宙 · 项目提案

> Casual Hackathon · Z.AI 赛道

---

## 一、问题

链上 IP（NFT 藏品、Meme 代币、数字艺术作品等）在发行后普遍面临"IP 沉睡"困境：

- **缺乏持续内容运营**：项目方精力集中在 mint 阶段，后续内容产出断层
- **创作门槛高**：社区成员想参与衍生创作但缺乏专业技能（写作、分镜、设计）
- **风格不统一**：自发衍生内容质量参差不齐，难以与原始 IP 风格对齐
- **效率低下**：人工创作周期长，难以规模化

## 二、解决方案

**IP Weave** — 一个基于 GLM-5.1 的自治 AI Agent。

它以 **Perceive → Plan → Execute → Reflect → Iterate → Deliver** 的完整闭环，自主完成从"理解链上 IP"到"交付衍生内容"的全过程。

### 核心创新

| 创新点 | 说明 |
|--------|------|
| **自治 Agent 而非工具** | 不是"点一下生成"，而是 Agent 自主拆解任务、规划步骤、执行并检查 |
| **GLM-5.1 驱动核心决策** | 风格分析、任务规划、质量检查全部由 GLM-5.1 完成 |
| **风格一致性闭环** | 自动检测产出与 IP 风格的对齐度，低于阈值自动迭代修正 |
| **多模态产出** | 一次任务同时产出衍生故事、动画脚本、周边资产 |
| **Web3 原生** | 从读取链上数据开始到内容交付，深度绑定链上 IP |

## 三、目标用户

1. **NFT 项目方 / DAO** — 自动化 IP 内容运营，降低人力成本
2. **链上 IP 持有者** — 为自己的数字资产生成专属衍生内容
3. **Web3 内容创作者** — 获取灵感和素材，加速创作流程
4. **Meme 社区** — 为社区 IP 持续产出新鲜内容

## 四、技术实现

### 技术栈

| 组件 | 技术选型 |
|------|---------|
| LLM | **GLM-5.1**（Z.AI Coding Plan 海外版） |
| 图像生成 | **CogView-3**（智谱） |
| 链上交互 | web3.py（准备中） |
| API 协议 | OpenAI 兼容格式 |
| 输出格式 | HTML / Markdown / JSON |

### 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    IP Weave Agent                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  输入层: 链上 IP 地址 / 预设名称                              │
│         ↓                                                    │
│  感知层 (Perceive)                                           │
│  ├─ ChainReader — 读取链上元数据                              │
│  └─ StyleAnalyzer — GLM-5.1 提取风格指纹                      │
│         ↓                                                    │
│  规划层 (Plan)                                               │
│  └─ GLM-5.1 — 制定多步骤创作计划                              │
│         ↓                                                    │
│  执行层 (Execute)                                            │
│  ├─ GLM-5.1 → 衍生故事                                       │
│  ├─ GLM-5.1 → 动画分镜脚本                                   │
│  └─ GLM-5.1 + CogView-3 → 周边资产概念                       │
│         ↓                                                    │
│  反思层 (Reflect)                                            │
│  └─ GLM-5.1 — 风格一致性检查 + 质量评分                       │
│         ↓ (不通过 → 迭代)                                    │
│  交付层 (Deliver)                                            │
│  └─ HTML报告 / 原始文件                                      │
└─────────────────────────────────────────────────────────────┘
```
